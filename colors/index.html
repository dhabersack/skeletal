Colors

_colors.scss contains a list of colors you want to use in your projects with the defaults being: 

$color-blue:   #3581fc;
$color-gray:   #bdbdbd;
$color-green:  #23db2a;
$color-red:    #e33d27;
$color-violet: #904fff;
$color-yellow: #ffe226;

You then create named proxies for them in _config.scss: 

$color-action:       $color-green;
$color-alert:        $color-yellow;
$color-brand:        $color-yellow;
$color-debug:        rgba($color-red, 0.0625);
$color-error:        $color-red;
$color-highlight:    $color-yellow;
$color-link:         $color-blue;
$color-link-visited: $color-violet;
$color-neutral:      $color-gray;

For all but $color-debug, Skeletal generates lighter and darker versions in _setup.scss by creating additional variables. This puts the following default palette at your disposal:

  • $color-action-lightest
  • $color-action-lighter
  • $color-action
  • $color-action-darker
  • $color-action-darkest
  • $color-alert-lightest
  • $color-alert-lighter
  • $color-alert
  • $color-alert-darker
  • $color-alert-darkest
  • $color-brand-lightest
  • $color-brand-lighter
  • $color-brand
  • $color-brand-darker
  • $color-brand-darkest
  • $color-error-lightest
  • $color-error-lighter
  • $color-error
  • $color-error-darker
  • $color-error-darkest
  • $color-highlight-lightest
  • $color-highlight-lighter
  • $color-highlight
  • $color-highlight-darker
  • $color-highlight-darkest
  • $color-link-lightest
  • $color-link-lighter
  • $color-link
  • $color-link-darker
  • $color-link-darkest
  • $color-link-visited-lightest
  • $color-link-visited-lighter
  • $color-link-visited
  • $color-link-visited-darker
  • $color-link-visited-darkest
  • $color-neutral-lightest
  • $color-neutral-lighter
  • $color-neutral
  • $color-neutral-darker
  • $color-neutral-darkest
  • $color-debug

This setup allows you to easily adapt the mapping of colors to semantic names and automatically update all colors used throughout your site.
